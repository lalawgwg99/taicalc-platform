# TaiCalc 測試套件

## 概述

本目錄包含 TaiCalc 平台的自動化測試套件，確保系統的穩定性和正確性。

## 測試類型

### 構建驗證測試 (`build.test.ts`)

驗證 Next.js 構建過程的完整性和正確性。

**測試範圍：**
- 構建過程成功完成
- 無編譯錯誤或警告
- 生成必要的輸出文件
- TypeScript 類型檢查通過
- ESLint 代碼品質檢查
- 構建性能指標
- Next.js 配置有效性
- 路由和 bundle 分析

**對應需求：** 1.2 - 構建過程執行時系統應成功完成構建而不出現錯誤

## 執行測試

### 執行所有測試
```bash
npm test
```

### 執行構建驗證測試
```bash
npm run test:build
```

### 監視模式執行測試
```bash
npm run test:watch
```

### 生成測試覆蓋率報告
```bash
npm run test:coverage
```

## 測試配置

- **測試框架：** Jest
- **測試環境：** jsdom (用於 DOM 測試)
- **配置文件：** `jest.config.js`
- **設置文件：** `jest.setup.js`

## 測試結構

```
__tests__/
├── README.md          # 測試說明文件
├── build.test.ts      # 構建驗證測試
└── ...               # 其他測試文件
```

## 注意事項

1. 構建測試會實際執行 `npm run build` 命令，因此需要較長的執行時間
2. 測試會檢查 `.next` 目錄中的構建輸出文件
3. 所有測試都應該在乾淨的環境中執行
4. 測試失敗時會提供詳細的錯誤信息以便調試

## 持續集成

這些測試設計為可在 CI/CD 管道中自動執行，確保每次代碼變更都不會破壞構建過程。
